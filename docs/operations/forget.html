Operation: forget (internal, policy-triggered)

Forget is an internal retention operation. In code, its objective is to bound active memory by archiving topics whose salience is below a policy threshold, primarily when topic volume crosses soft limits.

Access and trigger

  • Not client-accessible: no public route directly runs forget.
  • Triggered by reasoner from events: ingest_complete, query_complete, cron, memory_pressure.
  • Execution path: Reasoner.run(event) -> Executor.run_forget_low_salience().
Current behavior: forget runs only when reasoner is invoked. In stock API code that means post-response callbacks from /v1/ingest and /v1/query; UI API and LLM/MCP tool writes do not auto-trigger forget.

Objective in code

Objective: deactivate low-value topics by setting archived=true and reducing their salience, so active retrieval focuses on higher-signal memory.

Visual flow (current)

Forget runs only under policy conditions, then archives low-salience topics.

Trigger conditions (current implementation)

Event Condition for forget execution
ingest_completeRun when total topic count >= Policies.topic_count_soft_limit.
query_completeRun when total topic count >= soft limit.
cronRun when total topic count >= soft limit.
memory_pressureAlways run (explicit pressure signal bypasses count gate).

Forget algorithm (current implementation)

  1. Load non-archived topics via list_topics_for_embedding_scan(include_archived=False).
  2. For each topic, compare salience to forget_salience_threshold.
  3. If salience < threshold, set archived=true and salience = salience * 0.5.
  4. Append action marker archived:{topic_id}.

Execution sequence

reasoner event
  -> Reasoner.run(event)
  -> if event + policy conditions satisfied
      -> Executor.run_forget_low_salience()
      -> scan active topics
      -> archive low-salience topics
  -> return forget_actions[] in ReasonerResult

Policy knobs that control forget

Policy Default Current role
topic_count_soft_limit500Primary gate for forget triggers on ingest/query/cron events.
forget_salience_threshold0.05Topic salience cut-off for archive decisions.
max_topics_for_forget_scan10000Defined in policy model but not actively enforced in current forget loop.

Operational implications

  • Archived topics are skipped by default in active retrieval scans.
  • Forget decisions are currently based only on topic salience, not recency or semantic redundancy.
  • Because query operation bumps salience on touched topics, frequent reads can protect topics from archival.
