Interfaces

Architecture

Same memory, different callers.

Application services, internal operators, and LLM tooling each get an interface suited to their job. Agent-facing paths stay limited to observations and context queries; heavier maintenance stays on operator surfaces. All routes share one underlying store.

Three audiences

Product builders

Observation-shaped writes and context queries for the features you ship to customers.

Operators

Broader inspection and maintenance for people running the system.

Model-native flows

Optional surfaces where a model forwards observations and questions through tools when that fits your safety model.
