Runtime architecture overview

MemState runs as a single FastAPI process and keeps all durable state in one embedded Kuzu database file. All API surfaces share the same in-process store handle, so state is immediately consistent across /v1, /api/ui, and /api/llm/chat.

Deployment and process model

  • Single service binary path: python -m memstate.api.cli launches Uvicorn with memstate.api.main:app.
  • Single process ownership: one OS process owns the listening socket and shared graph handle for a database path.
  • No external DB tier: Kuzu is embedded and opened from MEMSTATE_KUZU_PATH.
  • No separate queue/worker: post-response background work uses FastAPI BackgroundTasks.

Core components and responsibilities

Component Primary role Called by
api.main Route layer, health endpoints, auth dependency wiring, router mounting. HTTP clients
api.deps Cached singleton providers for settings, executor, reasoner, shared store. FastAPI dependency injection
core.Executor /v1 business semantics: ingest placements, staged query, salience bump. /v1/ingest, /v1/query
reasoner.Reasoner Rule-based maintenance (forget low salience, merge duplicate titles). Background tasks after ingest/query
store.GraphStore Kuzu Cypher facade for topic, field, edge, and vector operations. Executor, UI API, tool runner
llm.MemoryToolRunner Executes LLM tool calls against the same shared store. /api/llm/chat, MCP server

Request lifecycle diagrams

Ingest path (POST /v1/ingest)

Client -> FastAPI route -> Executor.ingest
  -> GraphStore writes (topic/fields/edges, embedding update)
  -> HTTP response returned
  -> BackgroundTasks callback -> Reasoner.run("ingest_complete")
      -> run_forget_low_salience? (policy threshold)
      -> run_revise_duplicates? (duplicate-title heuristic)

Query path (POST /v1/query)

Client -> FastAPI route -> Executor.query
  -> embed query text
  -> semantic candidate selection (vector KNN or cosine fallback)
  -> optional structural expansion (neighbors + fields)
  -> optional temporal expansion (history when explain=true)
  -> salience bump + topic history append per returned topic
  -> HTTP response returned
  -> BackgroundTasks callback -> Reasoner.run("query_complete")

Shared singleton model

Dependency cache guarantees all major surfaces use one shared store object and one path-scoped Kuzu handle.

Boundary and error behavior

  • Validation errors: ValueError becomes HTTP 400 through a global exception handler.
  • Auth errors: optional API key mismatch becomes HTTP 401 from verify_api_key.
  • Graph health failures: /health/graph returns 503 with path and operator hint text.
  • Background failures: reasoner failures do not change the already-sent ingest/query response status.
