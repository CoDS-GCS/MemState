Limits, query stages, and configuration

MemState behavior is controlled by two distinct configuration layers: Policies (code-level defaults used by executor/reasoner) and Settings (environment-driven runtime options).

Policies (core behavior defaults)

Policy field Default Behavior impact
max_field_history500Max revisions retained per field.
embedding_dimension384Expected vector size for embedding providers.
query_salience_bump0.1Added to topic salience when topic appears in query results.
forget_salience_threshold0.05Below this, reasoner can archive topic in forget flow.
topic_count_soft_limit500Threshold for more aggressive reasoner forget behavior.
max_topics_for_forget_scan10000Bound for forget-scan operations.
By default these policy values are code defaults, not direct environment variables. Override by constructing custom policy objects in integration code/tests.

Query stages

POST /v1/query accepts stage subsets from semantic, structural, and temporal (default is all three).

Stage What executor does Output effect
semantic Embeds query, runs vector KNN or cosine fallback over topic embeddings. Candidate ranking and similarity scores in topic bundles.
structural Loads topic neighbors (RELATED) and selected fields. Populates neighbors[] and field entries in response.
temporal Controls whether field history can be expanded. Full history only when temporal stage is enabled and explain=true.

Stage interaction diagram

query request
  -> build candidate topic ids
       semantic on  -> vector/cosine ranking
       semantic off -> topic_ids allowlist or embedding scan order
  -> for each candidate topic
       structural on -> neighbors + selected fields
       temporal on + explain=true -> full field history
       otherwise -> current field snapshot only
  -> salience bump and topic history append
  -> response summary_text + candidates[]

Practical limits and caveats

  • top_k bounds the number of candidate topics returned/processed in query response construction.
  • version_field ingest placement requires exactly one field and a valid topic_id.
  • Large nested json fields are allowed, but payload size affects memory/IO and API response latency.
