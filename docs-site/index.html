MemState official system documentation

MemState is a single-process memory service built on FastAPI + embedded Kuzu. It offers three access surfaces on one shared graph: /v1 for product memory semantics, /api/ui for low-level graph operations, and /api/llm/chat for tool-based natural language access.

This site is written as the official engineering reference for maintainers and integrators. Every behavior described here is mapped to the current codebase under src/memstate.

System at a glance

The three user-facing surfaces converge on one shared GraphStore and one embedded Kuzu file.

Source map

  • memstate.api.main: app creation, route registration, auth dependency, health endpoints.
  • memstate.api.deps: singleton wiring for Executor, Reasoner, and shared GraphStore.
  • memstate.core.executor: all /v1 ingest and query semantics.
  • memstate.store.graph_store and store.kuzu_adapter: Kuzu access layer and schema bootstrap.
  • memstate.reasoner.engine: event-based policy loop (forget + duplicate revise).
  • memstate.llm.chat_api and llm.tool_runner: routed chat and tool execution.
